# 웹 프론트엔드 요구사항 정리

## 1. 범위
- 대상: 사용자용 웹 프론트엔드 앱
- 제외: 백엔드 상세 구현, 웹 어드민, Flutter 앱 상세 구현

## 2. 기술 스택
- `React 18+ (Vite)`
- `TypeScript`
- `Tailwind CSS`
- `TanStack Query`

## 3. 서비스 핵심 규칙 (프론트 반영 필요)
- 로그인 방식: 닉네임 입력 기반 Mock 로그인
- 룰렛 참여: 유저별 `1일 1회`
- 일일 예산: 하루 총 `100,000p`, 소진 시 당첨 불가(꽝 처리)
- 100(40%), 300(30%), 500(20%), 1000(10%) 랜덤 지급
- 포인트 유효기간: 획득일로부터 `30일`
- 상품 구매: 보유 포인트로 결제

## 4. 화면별 요구사항
- `로그인`
- 닉네임 입력으로 로그인(Mocking)

- `홈(룰렛)`
- 룰렛 UI 제공
- 룰렛의 영역은 각 포인트의 영역이 확률에 따라 달라져야 함.
- 오늘 잔여 예산 표시
- 오늘 잔여예산은 주기적으로 업데이트 하며 만약 5퍼센트 미만으로 내려가게 될 경우 빨간 글씨로 깜빡 거림.

- `내 포인트`
- 포인트 목록 표시
- 각 포인트의 유효기간 표시
- 만료된 포인트 상태(`만료됨`) 표시
- `7일 내 만료 예정 포인트` 알림 표시

- `상품 목록`
- 구매 가능한 상품 목록 표시
- 내 포인트 기준 구매 가능 여부 표시

- `주문 내역`
- 내 주문 목록 조회/표시

## 5. 공통 필수 구현
- 룰렛 애니메이션
- 백엔드 API 연동
- 로딩 상태 처리
- 에러 상태 처리

## 6. 평가 기준 (웹 프론트엔드)
- UI 디자인 완성도보다 기능 정상 동작 여부를 중심으로 평가

## 7. 제출/배포 관련 (웹 프론트엔드 해당분)
- 사용자 웹 앱 배포 URL 제출 필수
- 프론트엔드/어드민 배포는 Vercel 사용 권장
